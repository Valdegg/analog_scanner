Add the search pipeline to generic_scanner.py:

1. `async def search_query(pw, endpoint: str, query: str) -> list[dict]` -- Build the search URL via build_generic_search_url, fetch the page using scanner.fetch_page with retry loop (same pattern as scanner.scan_device: MAX_RETRIES attempts, 5s sleep between retries, broad except Exception), parse results with scanner.parse_listings, filter out old listings with scanner.is_too_old, filter out listings with no image_url (per decision: skip listings with no photos), filter out listings where price_eur > GENERIC_MAX_PRICE_EUR or price_eur is None. Return the filtered list. Log: "Query '{query}': {N} results, {kept} after filtering".

2. `deduplicate_listings(all_listings: list[dict]) -> list[dict]` -- Take all listings from all queries. Deduplicate by listing URL (the `url` field from parse_listings). When duplicates exist, keep the first occurrence. Return deduplicated list. Log: "Deduplicated: {before} -> {after} listings ({removed} duplicates)".

3. Update `async def main()` to be the full search pipeline:
   - Validate BRIGHTDATA_ENDPOINT from .env
   - Accept optional CLI arg for max queries (like scanner.py accepts limit): `python generic_scanner.py [limit]`
   - Load schema, generate queries
   - Loop through queries with progress `[{i+1}/{total}]`, call search_query for each, collect all listings
   - PAUSE_SECONDS sleep between queries (import from scanner)
   - Deduplicate collected listings
   - Filter old listings
   - Save intermediate results to `results/generic_scrape_DATE.json` with structure: `{"scrape_date": iso_str, "total_queries": N, "total_listings": N, "listings": [...]}` where each listing dict includes the fields from parse_listings plus a `"source_query"` field tracking which query found it
   - Print summary: total queries, total listings before/after dedup, output path
