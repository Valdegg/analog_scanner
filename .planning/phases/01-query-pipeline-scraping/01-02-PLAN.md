Add detail page scraping functions to generic_scanner.py:

1. `async def scrape_detail_page(pw, endpoint: str, listing_url: str) -> dict | None` -- Navigate to the listing's detail page using Bright Data (NOT reusing fetch_page because we need the browser/page object to run JavaScript for image extraction). Pattern:
   - Connect via `pw.chromium.connect_over_cdp(endpoint)` in try/finally like fetch_page
   - Navigate to listing_url with NAV_TIMEOUT_MS timeout, wait_until="domcontentloaded", then wait_for_timeout(3000)
   - Extract hero image: Find the main listing image element. On Kleinanzeigen detail pages, the primary image is typically in a gallery container. Use selectors like `img#viewad-image` or `img.galleryimage-element` or the first large image in the gallery section. Get the `src` attribute for the highest-resolution version. If no image found, return None (skip listing per decision).
   - Download the image and encode as base64: Use `page.evaluate()` to fetch the image via JavaScript's `fetch()` + `FileReader` to convert to base64 data URL string. This avoids needing an HTTP client -- the browser does the fetch. Extract just the base64 string (strip the `data:image/...;base64,` prefix). Alternatively, if the JS approach is unreliable, use Python's `base64` module with `page.goto()` to the image URL to get the bytes. Choose whichever approach is more robust.
   - Extract full description: Select the description container (typically `#viewad-description-text` or `.ad-keydetails` area). Get inner text.
   - Return `{"image_base64": str, "full_description": str, "image_content_type": str}` or None if no image.

   Use retry logic (MAX_RETRIES, 5s sleep) wrapping the entire function body, same pattern as search_query. Log progress and errors.

2. Update `main()` to add a detail scraping phase after the search + dedup phase:
   - After deduplication, iterate through listings with progress logging `[{i+1}/{total}] Scraping detail: {listing_title[:50]}`
   - Call scrape_detail_page for each listing
   - If scrape_detail_page returns None (no image), remove the listing from results and log skip
   - If it returns data, merge the image_base64, full_description, and image_content_type into the listing dict
   - PAUSE_SECONDS sleep between detail page requests
   - Save the final enriched listings to `results/generic_scrape_DATE.json` (same format as Plan 01 but now listings include image_base64 and full_description)
   - Print final summary: "{N} listings enriched with detail page data, {skipped} skipped (no image)"

3. Add `import base64` to imports if needed for image encoding.

IMPORTANT: The detail page HTML structure of Kleinanzeigen.de is not fully known. The executor should inspect an actual detail page if possible by navigating to one of the listing URLs from search results.
